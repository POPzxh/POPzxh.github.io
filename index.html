You are allowed to choose exactly 1 element from each row to form an array. Return the Kth smallest array sum among all possible arrays.

Constraints:

  • m == mat.length
  • n == mat.length[i]
  • 1 <= m, n <= 40
  • 1 <= k <= min(200, n ^ m)
  • 1 <= mat[i][j] <= 5000
  • mat[i] is a non decreasing array.

思路一、堆

对于每一层维护一个size<=k的大顶对：到第i层的最小k个数

每一层遍历前一层i-1生成的堆和mat[i]所有的组合。O(k*n)

很容易理解，如果第k个最小和是由第i行最小k+1个数衍生而来，那么对于第i+1层，最好情况是取其最小值，但是，第i层前k个数必定能产生至少k个数<k+1生成的，所以，每一层只需要维护大小为k的大顶堆。

代码

复杂度：O(m * n * k * log(k))：行数×（每一行的匹配数n×k）×更新pq（logk）

思路二、二叉搜索

我们先确定左右边界，即最小和与最大和，然后二分得到 mid，每次判断和小于mid的组合有多少个，如果大于等于 k那么更新 right，否则更新left

We want to select three indices i, j and k where (0 <= i < j <= k < arr.length).

Let’s define a and b as follows:

  • a = arr[i] ^ arr[i + 1] ^ ... ^ arr[j - 1]
  • b = arr[j] ^ arr[j + 1] ^ ... ^ arr[k]

Note that ^ denotes the bitwise-xor operation.

Return the number of triplets (i, j and k) Where a == b.

  • 1 <= arr[i] <= 10^8

思路

需要知道异或XOR的一个性质：

如果 a == b, 那么
a ^ a = b ^ a

然后，根据题意：

a = arr[i] ^ arr[i + 1] ^ ... ^ arr[j - 1]

b = arr[j] ^ arr[j + 1] ^ ... ^ arr[k]

对于i，k间所有的j是一样的，所以不需要考虑j，只需要找到满足的i、k对

  • 我们先求出0～i的XOR值存放在prefix[]中

如果，有一对i，k满足，那么prefix[i] ^ prefix[i-1]== prefix[k] ^ prefix[i]

显然，prefix[i-1] == prefix[k]，现在就变成了prefix中有多少组相同的值。
